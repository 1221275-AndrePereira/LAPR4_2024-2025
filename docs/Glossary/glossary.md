# Shodrone - Glossary of Terms

This glossary defines key terms used in the Shodrone project, referencing both the project requirements and potential code implementations.

| Term                            | Description                                                                                                                                                                                                                                                           | Code Implementation Hint / Related Classes                                                                                                                               |
|---------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **User & Roles** |                                                                                                                                                                                                                                                                       |                                                                                                                                                                          |
| User (System User)              | An individual with access to the Shodrone system, identified by a unique email from Shodrone's domain (`showdrone.com`). Authenticates to use system functionalities. [cite: Sem4PI_Project_Requirements_v02b.pdf, p. 9]                                             | Likely `eapli.framework.infrastructure.authz.domain.model.SystemUser` (from EAPLI framework). Associated with `ShodroneRoles`.                                         |
| Role                            | Defines the permissions and access rights of a System User within the backoffice application (e.g., Admin, CRM Manager, CRM Collaborator, Show Designer, Drone Tech). [cite: Sem4PI_Project_Requirements_v02b.pdf, p. 7, 9]                                            | `eapli.shodrone.usermanagement.domain.ShodroneRoles` (enum)                                                                                                              |
| Admin                           | A System User role with privileges to manage other users (register, disable/enable, list). [cite: Sem4PI_Project_Requirements_v02b.pdf, p. 7, 13]                                                                                                                       | `ShodroneRoles.ADMIN`                                                                                                                                                    |
| CRM Manager                     | A System User role responsible for managing customers (register, change status), proposal templates, and potentially decommissioning figures. [cite: Sem4PI_Project_Requirements_v02b.pdf, p. 7, 9, 14, 15, 18]                                                         | `ShodroneRoles.CRM_MANAGER`                                                                                                                                              |
| CRM Collaborator                | A System User role that interacts with customers, registers show requests, creates show proposals, and manages customer representatives. [cite: Sem4PI_Project_Requirements_v02b.pdf, p. 7, 9, 10, 13, 17]                                                              | `ShodroneRoles.CRM_COLLABORATOR`                                                                                                                                         |
| Show Designer                   | A System User role responsible for designing new figures, adding them to the catalogue, managing figure categories, and validating figure descriptions (DSL). [cite: Sem4PI_Project_Requirements_v02b.pdf, p. 7, 10, 14, 15, 18]                                        | `ShodroneRoles.SHOW_DESIGNER`                                                                                                                                            |
| Drone Tech                      | A System User role responsible for drone configuration (models, languages), inventory management, and testing/simulating figures and shows. [cite: Sem4PI_Project_Requirements_v02b.pdf, p. 7, 11, 14, 15, 18, 20]                                                      | `ShodroneRoles.DRONE_TECH`                                                                                                                                               |
| **Customer & Representatives** |                                                                                                                                                                                                                                                                       |                                                                                                                                                                          |
| Customer (Client Entity)        | A corporate or public entity in Europe that requests drone shows. Has a unique VAT number, address, phone number, and status (e.g., Regular, VIP). Represented in the system by `ShodroneUser`. [cite: Sem4PI_Project_Requirements_v02b.pdf, p. 9]                     | `eapli.shodrone.shodroneusermanagement.domain.ShodroneUser`. Status managed by `ShodroneUserStatus`, priority by `ShodroneUserPriority`.                               |
| Customer Representative         | An employee or contact person of a Customer (Client Entity) who interacts with Shodrone via the Customer App. Has a name, company email, phone number, and position. Each representative is a system user for the Customer App. [cite: Sem4PI_Project_Requirements_v02b.pdf, p. 8, 9, 13] | `eapli.shodrone.customerRepresentative.domain.CustomerRepresentative`. Associated with a `SystemUser` for Customer App access. `CompanyEmail` value object.           |
| Address                         | The physical address of a Customer (Client Entity). [cite: Sem4PI_Project_Requirements_v02b.pdf, p. 9]                                                                                                                                                                 | `eapli.shodrone.shodroneusermanagement.domain.Address` (Value Object for `ShodroneUser`)                                                                                 |
| Company Email                   | The email address of a Customer Representative, typically associated with their company. [cite: Sem4PI_Project_Requirements_v02b.pdf, p. 9]                                                                                                                               | `eapli.shodrone.customerRepresentative.domain.CompanyEmail` (Value Object for `CustomerRepresentative`)                                                                |
| Customer Status                 | The status of a Customer (Client Entity), e.g., Created, Regular, VIP, Infringement, Deleted. Managed by CRM Managers. [cite: Sem4PI_Project_Requirements_v02b.pdf, p. 9]                                                                                                | `eapli.shodrone.shodroneusermanagement.domain.ShodroneUserStatus` (Enum for `ShodroneUser`)                                                                              |
| Phone Number                    | Contact phone number for a Customer (Client Entity) or a Customer Representative.                                                                                                                                                                                     | `eapli.shodrone.shodroneusermanagement.domain.PhoneNumber` (Value Object)                                                                                                |
| VAT Number                      | A unique European Value Added Tax identification number for a Customer (Client Entity). [cite: Sem4PI_Project_Requirements_v02b.pdf, p. 9]                                                                                                                                 | `eapli.shodrone.shodroneusermanagement.domain.VATNumber` (Value Object for `ShodroneUser`)                                                                               |
| **Figures & Shows** |                                                                                                                                                                                                                                                                       |                                                                                                                                                                          |
| Figure                          | A composition of drone movements and formations, forming part of a show. Described by DSL code, has a version, description, exclusivity status, designer, and category. [cite: Sem4PI_Project_Requirements_v02b.pdf, p. 7, 9-10]                                         | `eapli.shodrone.figure.domain.Figure`. Identified by `FigureId`.                                                                                                         |
| Figure Catalogue                | A library of available figures that can be selected for shows. CRM Collaborators can list and search it. Show Designers manage its content. [cite: Sem4PI_Project_Requirements_v02b.pdf, p. 7, 13-14]                                                                   | Managed through `FigureRepository` and related application services.                                                                                                     |
| Figure Category                 | A classification for figures (e.g., "Logos", "Abstract Patterns"). Has a unique name, creation date, last update date, and status (active/inactive). [cite: Sem4PI_Project_Requirements_v02b.pdf, p. 14-15]                                                              | `eapli.shodrone.figurecategory.domain.FigureCategory`. Attributes like `FigureCategoryName`, `FigureCategoryStatus`, etc. Managed by `FigureCategoryCatalogue`.          |
| Figure DSL (Domain Specific Language) | A high-level, neutral language used to describe drone figures (geometric shapes, 3D bitmaps, movements like rotation/translation). Versioned. To be processed by ANTLR. [cite: Sem4PI_Project_Requirements_v02b.pdf, p. 7, 10, 15, 22]                             | Specification in LPROG. Parsed using ANTLR. Stored as part of the `Figure` entity.                                                                                        |
| Show                            | A sequence of figures performed by drones. Generated from a Show Proposal. [cite: Sem4PI_Project_Requirements_v02b.pdf, p. 9]                                                                                                                                            | Conceptual entity, likely composed of a list/sequence of `Figure` instances and drone configurations.                                                                  |
| Show Request                    | A customer's initial request for a drone show. Includes customer details, place, time, tentative number of drones, duration, and a description of desired figures/sequences. [cite: Sem4PI_Project_Requirements_v02b.pdf, p. 10, 13]                                    | `eapli.shodrone.showrequest.domain.ShowRequest`. Attributes like `RequestPlace`, `RequestShowDate`, `RequestNDrones`, `RequestDuration`, `RequestCreationDate`.            |
| Show Proposal                   | A formal offer from Shodrone to a customer, based on a Show Request. Includes figures, drone list, pricing (excluded from scope), and potentially a simulation video. Generated using templates. [cite: Sem4PI_Project_Requirements_v02b.pdf, p. 10-11, 17-18]            | Conceptual entity, leading to a `Show`. Workflow involves creation, data filling, figure addition, validation, and sending to customer. Uses `ShowProposalPlugin`.        |
| **Drones & Inventory** |                                                                                                                                                                                                                                                                       |                                                                                                                                                                          |
| Drone                           | An unmanned aerial vehicle used in shows. Each drone has a serial number, model, and runs specific routines for figures. [cite: Sem4PI_Project_Requirements_v02b.pdf, p. 7, 9]                                                                                             | Conceptual entity in the domain, physical asset. Represented in inventory.                                                                                               |
| Drone Inventory                 | A record of all drones owned by Shodrone, detailing their model, serial number, and status. [cite: Sem4PI_Project_Requirements_v02b.pdf, p. 14]                                                                                                                           | Managed by Drone Techs through specific application services.                                                                                                            |
| Drone Model                     | Specifies the type of drone, including its capabilities and behavior under environmental conditions (e.g., wind tolerance). [cite: Sem4PI_Project_Requirements_v02b.pdf, p. 10, 14]                                                                                         | Domain entity, likely `DroneModel`, with attributes for wind tolerance.                                                                                                  |
| Drone Programming Language      | The specific language used to program a drone model. Can be configured per model. Validated by a Drone Language Plugin. [cite: Sem4PI_Project_Requirements_v02b.pdf, p. 11, 15, 18]                                                                                         | Configuration setting associated with `DroneModel`.                                                                                                                      |
| Serial Number                   | A unique identifier for an individual drone. [cite: Sem4PI_Project_Requirements_v02b.pdf, p. 14]                                                                                                                                                                         | Attribute of a drone instance in the inventory.                                                                                                                          |
| **Simulation (SCOMP)** |                                                                                                                                                                                                                                                                       |                                                                                                                                                                          |
| Collision                       | An event where two or more drones occupy the same elemental volume (position) at the same time during a simulation. [cite: Sem4PI_Project_Requirements_v02b.pdf, p. 7, 11, 15-16]                                                                                        | Detected by the simulation engine.                                                                                                                                       |
| Elemental Volume                | A cubic unit of space within the 3D simulation environment. Used for collision detection. [cite: Sem4PI_Project_Requirements_v02b.pdf, p. 11]                                                                                                                             | Data structure element within the C-based simulation.                                                                                                                    |
| Orchestrator/Maestro Server     | A central server that coordinates drones during a full show simulation, giving orders to initiate figures and receiving feedback. [cite: Sem4PI_Project_Requirements_v02b.pdf, p. 7]                                                                                      | Component in the overall system architecture for show simulation.                                                                                                        |
| Shared Memory (Sprint 3 Sim.)   | A segment of memory used for inter-process communication between the multithreaded parent simulation process and child drone processes in the Sprint 3 simulation. [cite: Sem4PI_Project_Requirements_v02b.pdf, p. 18, 22]                                                | C-level IPC mechanism.                                                                                                                                                   |
| Simulation Report               | A document summarizing the results of a figure or show simulation, including drone count, execution status, collision details (timestamps, positions), and pass/fail validation. [cite: Sem4PI_Project_Requirements_v02b.pdf, p. 7, 11, 16, 19]                           | Generated file, output of the simulation process.                                                                                                                        |
| **Plugins & Integration** |                                                                                                                                                                                                                                                                       |                                                                                                                                                                          |
| ANTLR                           | A parser generator tool used for processing structured text, specifically for the Figure DSL and drone programming languages. [cite: Sem4PI_Project_Requirements_v02b.pdf, p. 12, 22]                                                                                     | Tool used in LPROG context.                                                                                                                                              |
| DSL Plugin                      | A software component responsible for analyzing and validating the Figure DSL. (US340, US341) [cite: Sem4PI_Project_Requirements_v02b.pdf, p. 12, 18]                                                                                                                      | External module integrated into the Backoffice Application.                                                                                                              |
| Drone Language Plugin           | A software component for analyzing and validating the syntax of code for a specific drone programming language. (US345, US346) [cite: Sem4PI_Project_Requirements_v02b.pdf, p. 12, 18]                                                                                      | External module integrated into the Backoffice Application.                                                                                                              |
| Show Proposal Plugin            | A software component used to format and generate the show proposal document based on a template. (US347) [cite: Sem4PI_Project_Requirements_v02b.pdf, p. 12, 18]                                                                                                           | External module integrated into the Backoffice Application.                                                                                                              |
| **Project Management & Process**|                                                                                                                                                                                                                                                                       |                                                                                                                                                                          |
| Sprint                          | A time-boxed iteration in the Scrum development process during which a specific set of work (User Stories) is completed. [cite: Sem4PI_Project_Requirements_v02b.pdf, p. 3-4]                                                                                              | Project management concept.                                                                                                                                              |
| User Story (US)                 | A description of a software feature from an end-user perspective. Used to define sprint backlog items. [cite: Sem4PI_Project_Requirements_v02b.pdf, p. 4, 12-20]                                                                                                        | Project management artifact.                                                                                                                                             |
| Continuous Integration (CI)     | An automated process of building the software and running tests whenever code changes are committed to the repository. (NFR05) [cite: Sem4PI_Project_Requirements_v02b.pdf, p. 12, 21]                                                                                   | Development practice, implemented using GitHub Actions/Workflows.                                                                                                        |
| Test-Driven Development (TDD)   | A software development approach where tests are written before the actual code. (NFR03) [cite: Sem4PI_Project_Requirements_v02b.pdf, p. 21]                                                                                                                            | Development methodology.                                                                                                                                                 |
| Non-Functional Requirement (NFR)| A requirement that specifies criteria used to judge the operation of a system, rather than specific behaviors (e.g., performance, security, usability). [cite: Sem4PI_Project_Requirements_v02b.pdf, p. 21-22]                                                            | Project requirement category.                                                                                                                                            |
| **Miscellaneous** |                                                                                                                                                                                                                                                                       |                                                                                                                                                                          |
| Backoffice Application          | The main server-side application used by Shodrone employees (Admin, CRM, Show Designers, Drone Techs) to manage the system. [cite: Sem4PI_Project_Requirements_v02b.pdf, p. 12, 17]                                                                                         | Core Java application. Main class likely `ShodroneBackOffice`.                                                                                                           |
| Customer App                    | A client-facing application (presumably simpler) used by Customer Representatives to interact with Shodrone (e.g., analyze proposals, check show agenda). [cite: Sem4PI_Project_Requirements_v02b.pdf, p. 9, 13, 17, 19-20]                                              | Separate application, potentially `shodrone.app.customer.console`. Interacts with `CustomerAppServer`.                                                                   |
| Customer App Server             | A server component within the Backoffice that handles requests from the Customer App. [cite: Sem4PI_Project_Requirements_v02b.pdf, p. 17]                                                                                                                                 | Part of the Backoffice Application, communicates via `CAS_Socket`.                                                                                                       |
| Elemental Volume                | A cubic unit of space in the 3D simulation grid used to detect drone collisions. [cite: Sem4PI_Project_Requirements_v02b.pdf, p. 7, 11]                                                                                                                                 | Concept in the SCOMP simulation part.                                                                                                                                    |
| Maintenance (Drone)             | Procedures to keep drones in operational condition. Information like type, date, and description is stored.                                                                                                                                                             | Attributes related to drone instances in the inventory.                                                                                                                  |
| Proposal Template               | A predefined structure or layout for generating show proposal documents. Can be configured by CRM Managers. [cite: Sem4PI_Project_Requirements_v02b.pdf, p. 15, 17, 18]                                                                                                    | Configuration for the `ShowProposalPlugin`.                                                                                                                              |
